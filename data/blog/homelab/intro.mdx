身為一名軟體工程師，日常生活除了寫程式之外，折騰家裡的網路和伺服器架構（HomeLab）也是一大樂趣。一個穩定、安全且易於維護的家庭網路環境，不僅能存放珍貴的攝影作品，還能流暢地運行各種自建服務。

今天這篇文章，我將帶大家了解我目前的家庭網路與伺服器架構，以及我是如何配置這些核心元件的。

## 網路與伺服器架構總覽

在深入細節之前，我們先用一張架構圖來了解整體的運作流程：

這套架構的核心理念是**「服務容器化」**與**「內網不穿透的安全存取」**。我不直接對外開放任何本地端的 Port，而是透過更現代化的方式來管理外部連線。

## 核心基礎設施 (Infrastructure)

### 1. 儲存與底層系統：TrueNAS

所有資料的最終歸宿與底層的作業系統，我選擇了 **TrueNAS**。它提供了強大的 ZFS 檔案系統，確保資料的完整性與容錯能力。無論是個人的文件、程式碼備份，還是龐大的 RAW 檔照片庫，交給 TrueNAS 都能讓人感到非常安心。

### 2. 容器化管理：Docker & Portainer

在服務的運行上，我全面採用了 Docker 容器化技術。這讓每個服務的環境各自獨立，升級或遷移都非常方便。
為了降低命令列管理的繁瑣度，我部署了 **Portainer** 來作為 Docker 的圖形化管理介面。透過 Portainer，我可以直覺地監控各個 Container 的資源佔用、快速重啟服務，或是透過 Web 介面更新 Image。

## 網路存取與安全配置 (Networking & Security)

### Cloudflare Tunnels：零信任的外部連線

這是我家中網路設定中最關鍵的一環。傳統的作法是在路由器上設定 Port Forwarding（通訊埠轉發）搭配 DDNS 來讓外部存取內網服務，但這會將家中的 IP 暴露在公開網路上，增加被攻擊的風險。

為了解決這個問題，我導入了 **Cloudflare Tunnels (cloudflared)**。

- **運作原理**：我在 Docker 中跑一個 `cloudflared` 的容器，這個容器會主動向 Cloudflare 的邊緣節點建立一條安全的加密隧道。
- **優勢**：當我在外地（例如從台北通勤或在日本旅遊時）想要連回家中查看服務，我的連線會先連到 Cloudflare，再由 Cloudflare 透過這條隧道將請求轉發給內網的機器。**家中路由器完全不需要開啟任何對外 Port**，大幅提升了網路安全性，同時也自帶了 SSL 憑證與 DDoS 防護。

## 自建服務亮點 (Hosted Services)

在這套穩固的網路基礎上，我目前運行了幾個主力服務：

- **Homepage**：作為所有自建服務的入口導航頁，一目了然地展示伺服器狀態、網路延遲與各個服務的快速連結。
- **Immich & PhotoPrism**：身為攝影愛好者，我需要強大且私密的相簿管理工具。利用這兩個服務搭配 TrueNAS 的大容量儲存，完美取代了 Google Photos，不僅能備份手機照片，也能有效管理相機的高畫質圖檔。

## 結語

這套基於 TrueNAS + Docker + Cloudflare Tunnels 的架構，在「便利性」與「安全性」之間取得了很好的平衡。接下來我也會陸續在部落格中分享個別服務（如 Immich 的轉檔最佳化、Homepage 的進階設定）的詳細配置心得，歡迎持續關注！
